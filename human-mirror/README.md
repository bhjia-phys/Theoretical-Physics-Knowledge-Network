# Human Mirror

This directory is the public, human-facing mirror layer for the repository.

Rules:

- `human-mirror/` is not the canonical backend.
- Canonical reusable knowledge still lives in `sources/`, `units/`, and `edges/`.
- Object-like mirror notes should carry `kb_unit_id` or `kb_source_map_id`.
- Structural control notes may carry `kb_surface`.
- The mirror may add explanation, intuition, examples, and navigation, but should not erase typed ids, source grounding, assumptions, regime, or dependency structure.

The current mirror is a deliberately small example rather than a full vault export.
